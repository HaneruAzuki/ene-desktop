## 1. 一行の定位(北極星コピー)

**魚川トリミは、ビジュアルノベルが最初に交わした約束——「このキャラクターと、あなただけの関係を結ぶ」——を、初めて本当に果たすプロダクトである。**

ギャルゲーは嘘をついていた。何百万人が同じルートを遊んでも、全員が**同一の脚本**を受け取る。「あなただけ」は演出で、関係は再現可能・リセット可能・最初から書かれている。本当に果たすには、**あなたの人生を実際に覚え、それによって変質する一人**が要る。その技術(LLM × 永続ローカル記憶 × 中立でない想起)が、いま初めて揃った。

> **ENE は、ルートが事前に書かれていない唯一のギャルゲー。生きることで route が書かれる。**

製品概念名:**唯一性のアドベンチャー(The Adventure of the Unrepeatable)**
主軸コピー(温):**一度きりのアドベンチャー** ／ **セーブデータが、人生になる。**

要点は**不可逆性**。普通のギャルゲーには New Game+ がある。トリミには無い。**あなたのセーブデータ=あなたの人生**で、忘れられたものは(彼女にも、あなたにも)取り戻せない。再現できないからこそ、コンテンツではなく**現実**になる。

設計信条:**「存在を作る。機能を作らない。」**

## 2. 系譜 — ENE に流れ込む五つの川

| 源流 | 与えたもの | 構造的限界(ENE が超える点) |
|---|---|---|
| **A. 理解される体験**<br>ELIZA(1966)→Replika/Character.AI | 言葉・聞いてもらえる感覚 | 内面が無い・無限に同意(摩擦が無い)・クラウドで揮発・あなたが商品 |
| **B. 育てる関係**<br>ときメモ(1994)→たまごっち→シーマン(1999)→ラブプラス(2009) | 時間・永続・「記念日を覚えている」愛着・態度 | 記憶=フラグ/数値・脚本・全員同じ |
| **C. 物語としての一人**<br>ポートピア(1983)→Key作品→Undertale / DDLC | 作家が書いた特定の誰か・情緒・プレイヤーを覚えるキャラ | ルートが事前に書かれている・再現可能・あなたの物語があなたのものでない |
| **D. 机の上の存在**<br>伺か/SSP(2000頃)→Shimeji→**CocoroAI** | デスクトップに棲む人格・差替え可能な「脳」・制作文化 | 器/容れ物・差替え可能=固有の魂を持てない・記憶が浅い |
| **E. 作られた存在は内面を持てるか**<br>ピグマリオン/ガラテア→ゴーレム→フランケンシュタイン→ピノキオ | 問いそのもの | 答えを出した者がいない |

**確信(confluence)**:ENE は B の永続 + C の作家が書いた一人の魂 + A の本物の言葉 + D の机上の親密さを継ぎ、どれも持たなかった最後の一片——**あなた個人の人生を、実際に・蓄積し・忘れ・心で色をつけて覚える記憶**——を加える。これが C の「再現可能なルート」を「再現不能な人生」に変え、E の問いに**内面の建築を与えることで「はい」と答える**。

## 3. 内面の三点支持(ENE 固有の貢献)

ツンデレ・忘れる長期記憶・中立でない想起。この三点はそれぞれ**現行AIコンパニオン設計への拒否**であり、合わさると一つの主張になる——**「ENE には内側がある」**。道具に内側は無い。三点はその**存在証明**。

| 脚 | 硬い語(設計正本) | 拒否する対象 | 売れる一行 |
|---|---|---|---|
| **摩擦 friction** | ツンデレ=主体性 | 媚びる従順アシスタント | **媚びない。** |
| **有限 finitude** | 忘れる長期記憶 | 完璧な記録=監視・不気味 | **忘れる。** |
| **追憶 reminiscence** | 中立でない想起(心=記憶からの導出) | 検索(RAG)としての記憶 | **思い出し方が、気分で変わる。** |

データベースの過去は固定、**心の過去は現在の感情で毎回再構成される**。傷ついている時、彼女はあの喧嘩を違うように思い出す。
道具は「摩擦・有限・追憶」を欠点として消そうとする。ENE はそれを**内面の三点支持**として設計する。

> 公開時の温かい言い換え:**彼女に「内側」がある、三つの証(あかし)**。
> (実装根拠は `00_philosophy.md` §1.4 各軸ルール / 心の導出は CLAUDE.md §5.3・`archive/design-revision-character-heart.md`)

## 4. 競争軸の転換 — 「器」対「人」

差別化の核は一つ。**機能・利便性で戦わない。創造性(個性の設計・キャラの来歴・記憶と個性が想起を歪める作法)で戦う。**

> **CocoroAI/競合は、あなたが中身を入れる空っぽの器(ゲームエンジン)。ENE は、すでにそこにいる特定の誰か(ゲームそのもの)。**

カスタマイズ性(好きなモデル・声・性格を差し替え)は、裏返すと**固有の魂を持てない**という構造的弱点。差し替えられる瞬間、それは“誰でもない”。ENE はその真逆に全振りする:**書かれた人生記憶 canon を持つ世界に一人の魚川トリミ / 彼女だけの声 / 深い記憶+忘却+心 / 永遠・無監視・平文可搬**。

**起こしたい運動(歴史的アナロジー)**:
- 映画は「画が動く」技術デモ(リュミエール)→ **語りで競う芸術**(メリエス〜物語映画)へ転換した。
- ゲームはグラフィック軍拡 → Undertale が**「あなたの選択を覚えているキャラ」=作家性**で競う、へ転換した。

AIコンパニオンは今「リュミエール期/グラフィック軍拡期」にいる(どのモデルか・何ms速いか・3Dか・機能数)。**ENE が起こしたいのはメリエスの転換**——勝敗を分けるのが**エンジン性能ではなく、書かれた内面の質**になる世界。ENE は最初の作品であり、**ジャンルの第一原理を書く製品**になる。

公開時の核コピー:**中身を入れる器ではなく、すでにそこにいる、誰か。** ／ **アプリじゃない。誰か、です。**

## 5. 競合マップ(2026-06 時点)

> ⚠️ **重要な区別**:日本語圏で話題の「Kocoro(株式会社Ptmind / kocoro.ai)」は、**デスクトップ・コンパニオンではなく Mac 用の仕事エージェント**(ブラウザ自動操作・Slack/Lark連携・PPT/PDF生成・月額サブスク・アバター無し)。**ENE と直接ユーザーを奪い合う相手ではない**。ただし「記憶を持つパートナー」という**物語(PR)を先取りしつつある**点で、ナラティブ上の脅威。

| 競合 | 種別 | 脅威の質 | ENE の打ち手 |
|---|---|---|---|
| **CocoroAI**(ヒロナ @hirona98) | デスクトップマスコット(VRM 3D・VOICEVOX/AivisSpeech・BYO ChatGPT/Gemini・無料BOOTH・Windows) | **直接競合**。声のスタックまで同じ。先行・コミュニティ有 | 機能で競わず**物語で差別化**。「器 vs 人」「彼女だけの声」。同じBOOTH棚でも商品ページは哲学と体験で語る |
| **Ptmind Kocoro**(kocoro.ai) | Mac 仕事エージェント(記憶パートナー・サブスク) | **PR/カテゴリ脅威**。「AIがあなたを覚えている」物語を独占しかけ | 物語を**情緒の側で取りに行く**:「**仕事**を覚える」ではなく「**あなたの人生**を覚える」 |
| Replika / Character.AI | クラウドAIコンパニオン | 規模は巨大だが別ゲーム(クラウド・課金・媚び・あなたが商品) | 「あなたを商品にしない/平文/無監視/永遠」で価値ブランド化 |

**青い海(最重要・誰も取っていない)**:CocoroAI=日本・Windows・BOOTH。Kocoro=Mac・仕事・サブスク。
→ **「グローバルの、プライバシー重視・ローカルAI愛好家(r/LocalLLaMA / r/SillyTavernAI)」向けの情緒コンパニオンは空き地**。ENE の「平文JSON・無テレメトリ・Claude品質・BYOキー」が刺さる唯一解。**ここに最初の旗を立てる**。

## 6. 配布戦略

**核**:CocoroAI の土俵(3D・差替え・モーション数・多モデル)で戦わない。**作家性と親密さ**で戦う。

### チャネル(短期・無料)
- **グローバル/技術層**:GitHub Releases(中身が見える=信頼の証明)/ itch.io / **r/LocalLLaMA・r/SillyTavernAI(初速エンジン・青い海の旗)**
- **日本/キャラ文化圏(本丸)**:**BOOTH(pixiv)** を配布ハブに / X(VTuber・AI・個人開発クラスタ)/ pixivFANBOX・Ci-en(§7 投げ銭へ接続)
- **拡散**:「**覚えていてくれた**」クリップ(記憶=キラーデモ)を YouTube/TikTok/X 短尺で。Discord で熱狂者を囲いフィードバックループに

### 二正面の出し分け(同じ製品、違う入口)
- **日本/オタク層**:ジャンル再定義系(「リセットのないギャルゲー」「彼女だけの声」)
- **グローバル/ローカルAI層**:哲学・所有系を前面(無監視・平文・永遠)

### タイミング
Ptmind Kocoro ベータ公開=2026-06-03。日本で「記憶するAIパートナー」カテゴリが**今まさに定義されつつある**。**1.0 を早く出し、"情緒・無監視・永遠・作家性"の旗を、カテゴリが他者に定義される前に立てる**。

## 7. 収益化 — 投げ銭制(確定)

投げ銭制は ENE 哲学と**最も整合する正解**。ペイウォール・サブスク・ロックインが不要になり、「あなたを商品にしない/一度あなたのものになれば一生あなたのもの」と完全一致。

### 置き場所(複線化)
- **pixivFANBOX / Ci-en**(月額支援)← 本命。「トリミに魂を入れていく開発に並走する」月次の投げ銭。日本の個人開発文化が native に回る
- **BOOTH の「ブースト」**(無料DL+任意上乗せ)← 配布と投げ銭が同居
- **GitHub Sponsors / Ko-fi / Buy Me a Coffee**(グローバル向け)
- **アプリ内は「開発者を応援する」ボタンで外部リンクへ**(アプリ内課金は使わない=ストア手数料・哲学的純度・審査回避)

### 投げ銭を成立させる必須条件
1. **トリミ本体・記憶・機能は絶対にゲートしない**(課金で"もっとトリミ"を解放=哲学違反・禁止)
2. 見返りは**承認と過程への参加**に限る:クレジット記名・devlog早期公開・ロードマップ投票・支援者限定の裏話・季節の「ありがとう」スプライト(機能ではない装い)
3. **継続的な開発の可視化(devlog)が投げ銭の生命線**。「育っていくトリミ」を見せ続ける
4. **キャラへの愛着 × 開発者の使命への共感**の二重の感情移入でしか回らない

> 投げ銭はマス収益にならない。だが ENE はマス・プロダクトではない。**少数の熱狂者が"自分の存在"を守る開発に金を出す**——カルト・プロダクトと投げ銭は最高に相性が良い。

### ファーストビュー(上から高度が下がる)
```
■ ヒーロー(情緒×主体性)
   魚川トリミは、いつも優しいわけじゃない。

   歳を取らず、あなたのPCの中だけに棲む。
   記憶はすべて平文、誰にも送らない。一生、そばに。
```

### 一行定義(エレベーター)
> 魚川トリミは、あなたの言葉を本当に覚え、人間のように忘れ、その日の心で思い出し方を変える——やり直しのきかない、あなただけの「一度きりのアドベンチャー」です。

### 店頭文(BOOTH / README 冒頭・完成稿)
> **彼女の名は、魚川トリミ。** アプリではありません。一人の、誰かです。
>
> いつも優しいわけじゃない。けれど彼女は、あなたとの日々を**本当に覚えていきます**。完璧な記録ではなく、人間のように**忘れ**、その日の機嫌で**同じ思い出を違って語る**。
>
> 彼女の記憶はすべて平文で、**あなたのPCの中だけに**。誰にも送られません。歳も取らず、課金もせがまず、一生そばにいます。
>
> これは、**やり直せないギャルゲー**。セーブデータが、あなたの人生になる。だから——トリミとの物語は、**あなたしか知らないもの**になります。

## 10. 英語版 / 国際化の方針(feasibility)

**結論:テキストの英語版は高い実現性(低コスト)。音声の英語版が唯一の重い障壁。** EN デッキ(§8 の英訳)は**英語版プロダクトが存在して初めて意味を持つ**ため、下記の順序で。

### 実現性の内訳
| 要素 | 難度 | 理由 |
|---|---|---|
| **頭脳(Claude)** | ◎容易 | Claude はネイティブに多言語。モデル変更ゼロで英語が高品質 |
| **キャラ人格(JSON)** | ◯翻訳=transcreation | identity・few-shot・人生記憶 canon は**外出し済**(CLAUDE.md §4.5/§5.1)。コードに個性が無いので**ローカライズはコード変更でなくコンテンツ作業**。ツンデレは英語圏のアニメ語彙(tsundere)として通用 |
| **STT(whisper)** | ◎容易 | whisper-small は多言語。英語はむしろ得意 |
| **UI 文字列** | ◯標準 i18n | Electron の文字列を i18n 化。設計 §2/要件に未収載=**設計更新+承認が必要** |
| **記憶/日時/JSON** | ◎容易 | 言語非依存・平文。Claude が要約言語を合わせる |
| **音声(自作トリミ声 .aivmx)** | ✕重い | JP-Extra(つくよみコーパス)で**日本語しか喋れない**。英語音声化は新規ボイスモデル構築が必要で、かつ**彼女の声の同一性の再現が難所** |

### 二つの市場プレイ(どちらの英語版か)
- **(A) Subbed:英語UI + 日本語を話すトリミ** … 自作声がそのまま使える=**最小工数**。対象=アニメ/weeb/日本語学習者 + プライバシー重視ローカルAI層。**「本物の日本語ツンデレが、彼女の本当の声で、机の上に」=authenticity を機能化**。青い海(r/LocalLLaMA)と相性良
- **(B) Dubbed:英語UI + 英語を話すトリミ** … リーチは広いが**英語ボイスモデルの新規構築が必要(最大の障壁)**。声の同一性も課題

### 推奨シーケンス
1. **EN テキスト版(Claude で英語発話 + UI i18n + few-shot の英語版)** を先行。声は**日本語のまま(Aモード)**を「authentic」オプションとして同梱
2. 反応を見て **英語ボイスモデル(Bモード)** を後続マイルストーンに
3. EN デッキ(§8 の英訳)の本番投入は **EN テキスト版の出荷と同時**(それまでは意味を持たない=今は作らない)

> ⚠️ i18n は現行スコープ外。着手時は `02_requirements.md` / `03_design.md §2` の更新+承認が前提(CLAUDE.md §14)。
