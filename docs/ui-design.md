## 0. 方針(なぜこの形か)

- **原則**:「ユーザに説明しなくてもわかる」× **キャラ性 > 機能性**(Vision §5.1)。トリミは“**存在**”であって“アプリ”ではない。
- **緊張関係**:発見可能性(ボタンを見せる)と、佇まいの維持(ボタンを並べない)は放っておくとぶつかる。
- **解法**:
  1. **待機中はトリミだけ**(画面に操作を出さない)。
  2. **キャラの上にホバーすると操作が現れ、離れると消える**。
  3. 操作は**キャラ下部の不透明部(胸元)にオーバーレイ**する。**透明な余白には何も置かない**。
     - 理由:余白(クリックスルー域)に操作を置くと、ボタンへ手を伸ばす途中で透明域に入り、**ボタンが消えてしまう**。キャラの不透明部に重ねればホバー領域が地続きになり、消えない。

## 1. レイアウト

```
┌──────────────────────┐
│        吹き出し(回答だけ)         │ ← 上はボタンを置かない
│                                  │
│        トリミ(バストアップ)         │ ← 現行の大きさを維持(縮小しない)
│   ┌──────────────────┐  │
│   │ [🎤][🔊][☕][⚙][✕] │  │ ← ホバー時、胸元に重ねて出す
│   │ ┌──────────────┐ │  │
│   │ │ 話しかける…(入力ピル)  │ │  │
│   │ └──────────────┘ │  │
│   └──────────────────┘  │
└──────────────────────┘
```

- **上**=応答吹き出し専用([03_design.md](03_design.md) §8.5)。操作ボタンは置かない。
- **キャラ**=現行のバストアップ表示・ウィンドウサイズを維持(§8.1)。**縮小しない**。
- **操作**=ホバー時にキャラ下部へオーバーレイ。横一列のボタン＋その下に入力ピル。
- **ボタン順(左→右)**:**マイク / 音量 / 離席 / 設定 / じゃあね**。
  - マイクのみ少し大きめ(主操作の強調)。じゃあねは少し離す(誤クリック防止)。最頻用のマイクが左端・終了系が右端で最も離れる。

## 2. 表示と当たり判定(クリックスルー)

- **ホバートリガ** = キャラの不透明部(or 吹き出し)。
- 操作はキャラ不透明部の上に出るので、**ホバー領域が地続き**=手を伸ばしても消えない。
- **非ホバー時は操作を即アンマウント**。これは「見た目をすっきりさせる」ためであり、クリックスルー復帰のためではない(キャラ域はもともと非透過なので、操作を消してもクリックスルー挙動は変わらない)。
- **常駐サイン**(非ホバーでも残す・例外):
  - **マイクON** = 緑点灯+「聞いてるよ」(「今、聞かれている」を常に可視化=プライバシー)。
  - **離席中** = キャラが後ろを向く。
- 当たり判定の配線は [use-click-through.ts](../src/app/renderer/use-click-through.ts) を更新し、合体領域(キャラ不透明部＋各ボタン＋入力ピル＋吹き出し)を不透過として扱う。

## 3. 各操作の仕様

### 入力ピル(テキスト)
- ホバーで表示。クリックでフォーカス→入力。**Enter 送信 / ESC 閉じる**。
- **フォーカス/最初の1打**で、Tier0 キャッシュが未ウォームなら温める(レイテンシ施策)。
- 旧「キャラを直接クリックで入力欄を開く」は**廃止**し、ピルに集約。

### ① マイク(単一ボタン・ハイブリッド)
- **OFF 時**:**短タップ=ハンズフリーON**(呼びかけたら聞く)/ **長押し=押している間だけ PTT**(押して話す)。
- **ON 時**:タップで OFF。
- ON 中は**緑点灯+「聞いてるよ」を常駐**。
- ホバーツールチップで「長押し=押して話す」を告知(長押しは隠し操作のため補う)。
- ⚠ 実装注意:短タップ/長押しの判別閾値(≈250ms)と、barge-in・相槌のエコーガードとの相互作用に注意。

### ② 音量
- ホバーで音量ノブ、クリックでミュート(対象=**トリミの声=出力**)。
- [audio-player.ts](../src/app/renderer/audio-player.ts) の再生グラフに `GainNode` を挟む。音量・ミュート状態は設定に永続化。

### ③ 離席
- クリックでトグル。離席中は:
  - キャラが**後ろを向く**(VRM=180°回頭 / PNG は後ろ向き素材が無いため**暗転＋「離席中」表示**で代替)。
  - **マイク OFF**。
  - **自発発話(idle-talk)を停止**(誰もいない椅子に話しかけない)。[idle-talk-manager.ts](../src/app/main/idle-talk-manager.ts) の `tick()` 冒頭で `runtime.away` を見て return。
- ホバー「ちょっとまってね」。

### ④ 設定
- クリックで**統合設定パネル**を開く(§4)。

### ⑤ じゃあね
- クリックで「**＼じゃあね／**」ポップ→**トレイにしまう＋マイク OFF**。
- **初回のみ**「トレイで待ってるよ」バルーン通知(`Tray.displayBalloon`・Windows)。以後出さない。
- **完全終了はトレイメニュー**から。
- 設計判断:常駐型コンパニオンの定石(Slack/Discord/Spotify 等)に倣い、主ボタンは「しまう」。トレイアイコン(常時可視)＋初回通知＋マイクOFF の3点で「閉じたつもり問題」を防ぐ。「一生そばにいる存在」(Vision §1)を誤クリックで失わせない。

## 4. 統合設定パネル

④設定ボタンから開く1枚。**機能名ではなく人間の言葉**でラベル付けする。

| 項目 | 内容 | 旧UIからの移行 |
|------|------|--------------|
| 話しかけてくる頻度 | しない / 控えめ / ふつう(`idleTalk`) | **新規UI**(従来は env/JSON のみ) |
| 見た目の調整 | VRM 表示(高さ・距離・向き・腕) | 旧:右上の歯車パネル → **内包** |
| API キーを変更 | API キーダイアログを開く | 旧:右クリックメニュー → 移設 |
| 位置をリセット | ウィンドウを既定位置へ | 旧:右クリックメニュー → 移設 |
| このアプリについて / クレジット | バージョン・音声クレジット | 旧:右クリックメニュー → 移設 |

- 旧「マイク入力方式」サブメニュー(右クリック)は、マイクのハイブリッド化により**廃止**。
- **右クリックメニューは廃止**(設定=ボタン、完全終了=トレイで全機能に到達できるため)。

## 5. 既存挙動(維持するもの)

- **移動**:キャラのドラッグでウィンドウ移動([03_design.md](03_design.md) §8.2/8.3)。
- **吹き出し**:30秒で自動消滅 / クリック / ESC で閉じる(§8.5)。
- **トレイ**:アイコンのクリックで表示/非表示、メニューに「ENE と話す / ENE について / ENE を終了」([tray.ts](../src/app/main/tray.ts))。

### 実装順序(各段で実機クリックスルー smoke を依頼)

1. **土台**:上=吹き出し専用 / 下=バー枠＋ピルをキャラ下部へオーバーレイ / 合体ホバー領域で fade-in・即非表示 / click-through refs 更新 → **実機確認**
2. **①マイク**:単一ハイブリッド(長押しPTT/タップトグル)＋ON常駐点灯
3. **②音量/ミュート**(`GainNode`＋永続化)
4. **⑤じゃあね**→トレイにしまう＋マイクOFF＋初回バルーン
5. **③離席**(後ろ向き/idle-talk停止/マイクOFF/常駐サイン)
6. **④設定パネル統合**(話しかけ頻度=新規UI / VRM内包 / APIキー / 位置リセット / クレジット)＋右クリック廃止

### リスク

- **クリックスルー回帰**(最重要)。本リポジトリの環境では実機確認ができないため、各段でユーザの smoke が必要。
- **PNG に後ろ向き素材が無い** → 離席の見せ方は暫定(暗転＋表示)。
- **長押し/タップ閾値**と barge-in/エコーガードの相互作用。
- `displayBalloon` は Windows 専用(本プロダクトは Windows 専用なので問題なし)。
