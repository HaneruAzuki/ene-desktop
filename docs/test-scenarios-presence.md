## 1. テストの3層と実行方法

| 層 | 何を確かめるか | 実行 | 判定 |
|---|---|---|---|
| **A. 純粋ユニット** | 各判定ロジック単体(時間帯/経過/気にかけ選択/ギャップ/発火判定/パース) | `npm test` | 自動(決定論) |
| **B. E2E 記憶シナリオ** | 「記憶を書く→プロンプトに正しく注入/除外されるか」の配線(全機構) | `npm test` | 自動(決定論) |
| **C. 実API behavior** | 「APIでしか分からない挙動」=モデルが規範に従うか/作話しないか | 下記(キー必要) | 自動チェック＋**Claude判定** |

**A・B は `npm test` に含まれ常時緑**(オフライン・ネットワーク無し・**無料**)。
B は2本:`presence-scenarios.test.ts`(存在感機能=いま/気にかけ/まだ知らないこと/誕生日/名前読み)＋
**`memory-regression.test.ts`(記憶機構=想起の語彙索引/supersede除外/開示ゲーティング/semantic全フィールド/短期記憶/想起件数上限)**。
「記憶を改修したら過去の記憶挙動が壊れていないか」はまずこの2本が**自動で**捕まえる(記憶バグの大半)。
**設計方針**:プロンプトに正しく入るかは B(無料)で見る。C(課金)には「APIでしか分からない挙動」だけを置く=最小コストで網羅。
**C は普段は skip**。実行するには API キーと明示フラグが要る(PowerShell):

```powershell
$env:ANTHROPIC_API_KEY = "sk-ant-..."   # 自分の鍵を一時的に
.\scripts\presence-live-check.ps1        # = ENE_LIVE_TEST=1 で tests/live を実行
```

- **入力**は `tests/live/scenarios.json`(編集可)。各シナリオは**重複しない別々の失敗モードを1つずつ**
  (閉世界/矛盾指摘/反同調/自己作話/知識の輪郭/AI自称/想起活用/provenance非混同)=**1コール×8・最小で網羅**。
  Tier0+fewshot はプロンプトキャッシュで初回のみ課金(以降キャッシュ読み)=安価。
- **出力**は `presence-live-results.md`(入力/記憶/期待(PASS)/危険信号(FAIL)/自動チェック/回答。`.gitignore` 済)。
- **判定の二段**:(1) 機械=空でない/**AI自称検知=ハードFAIL**(`detectAiSelfReference`)/回避語ヒット=フラグ。
  (2) **Claude判定**=この結果ファイルを Claude に見せると、各シナリオの期待・危険信号と回答を突き合わせて OK/NG を出す。
  最終的な「トリミらしいか」は人間/Claude 判定が本質(CLAUDE §9.2・成功基準8)。

## 3. 自動で守れること / 人間が確かめること

- **自動(A・B)で守る** = 「記憶の状態に応じて、正しい文脈・規範・ヒントがプロンプトに載るか」。
  記憶まわりの改修で**配線が壊れたら即座に赤くなる**のはここ。回帰の主戦場。
- **人間が確かめる(C・手動)** = 「実際のトリミの応答がそれらしいか」。
  プロンプトに材料が載っていても、モデルがどう応じるかは確率的=最終判断は人間(成功基準8)。

記憶のバグ調査の初手は **B(E2E)を流す** → 落ちた行が「どの記憶→どの注入」が壊れたかを指す。
B が全部緑なのに体感がおかしい場合は **C** でモデルの応答傾向を見る(規範/例文の調整へ)。

## 4. 既知の単純化・先送り(v1)

- **忘却 × 暮らしの断片**:`daily-life`(provenance:'self')は現状 forgetting の対象外(canon と同様に保持)。
  「平凡な日は薄れる」までは、forgetting を実運用で有効化する際に provenance を保ったまま縮退させる対応とセットで行う
  (forgetting は既定オフ・レビュー後有効化のため v1 では未着手)。`docs/optimization-backlog.md` 参照。
- **自発発話の音声**:実装済み(通常応答と同じ `speakResponse`→voice-chunk 経路)。push-to-talk(既定)は
  マイクが押下中のみ=自声を拾わず安全。ハンズフリーは相槌で実証済みのエコーガードを継承するが、
  **ハンズフリーでの自己トリガ有無は実機 smoke を一度推奨**(本環境で Electron/音声を動かせないため未検証)。
- **C(実API)は本環境では未実行**(キー無し)。ハーネスは ready-to-run。キーを入れて目視確認すること。
