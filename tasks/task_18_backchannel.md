# Task 18: 能動的リスニング(相槌・思考フィラーエンジン)

## 目的

ユーザの発話を聞きながら、**適切なタイミングで適切な相槌**(「うん」「へえ」「なるほど」＋うなずき等の非言語)を打ち、
さらに ENE が答える入りで**思考フィラー**(「うーん…」)を出すことで、**「自分の話を聞いてもらえている」感**を作る。
処理は**完全ローカル・超軽量・即時**(リアルタイム経路に Claude/ネットワークを置かない)。

> 本タスクは 2026-06-08 のユーザーとの設計セッションで確定。背景の議論は本ファイル「設計の憲法」と
> `docs/optimization-backlog.md` 冒頭の三原則を参照。

## 位置づけ・哲学

`docs/00_philosophy.md` ロードマップの **「能動性」次元**(製品1.0の本命)の最小実装。
相槌は ENE の応答ターンではなく、**ユーザをリッスンする側**の振る舞い。companionship そのもの。

## 設計の憲法(越えてはいけない三原則 ＋ 装置の地図)

**三原則**(`optimization-backlog.md` と共有):
1. **キャラの性格を言い訳に「遅い」を正当化しない**(計算は常に最小化)。
2. **遅延の利用は「速すぎる応答を一定の"間"まで意図的に遅らせる」だけ**(遅いのを引き伸ばさない)。
3. **相槌・フィラーは必要**。その時間は**棚からぼたもち**(技術的限界に時間を与える副次効果)であって、**設計目標にしない**。

**判別テスト** ―「もしアプリが一瞬で応答できたら、それでもこれをやるか?」
- 相槌 → YES(一瞬でも聞いてる合図は欲しい)→ 正当
- 思考フィラー → YES(熟考に値する問いなら)/ NO(全ターンに付けるなら=遅延隠し)
- 遅延隠しの埋め草 → NO → **禁止**

**装置の地図**(混同しない):

| 装置 | ターン | 例 | 正当化 | 本タスク |
|---|---|---|---|---|
| **相槌** | 聞く(ユーザの番) | うん/へえ/なるほど | 聞いてもらえてる感 | ★主目的 |
| **思考フィラー**(考える間の声) | 答える入り(熟考時) | うーん/そうだなぁ | 本当に考える価値がある問い | ★(Phase C・F-ANIM-04の音声ツイン) |
| 間(ため) | 答える | (無音の一拍) | 自然なテンポ | 対象外(別施策) |
| ~~遅延隠しの埋め草~~ | 答える | 何でも | (なし) | **禁止** |

## 重要な開発制約(2026-06-08 確定)

- **開発者は一人(ユーザー)で、調教(学習データ作成・反復チューニング)ができない。**
  → **Claude が本PC・本CPUで開発する。** 学習を要する設計は避け、**ルールベースを第一**にする。
  モデルを使う場合も **Claude が生成した合成データから本CPUで訓練**(GPU不要・ユーザーは訓練しない)。
- **ユーザーの唯一の関与は「自然か」の最終試聴判定**(成功基準8=人間判定はユーザーの領域・規約§9.3・manual-check方針)。
- リアルタイム経路は**完全ローカル**(脳もネットも使わない)=ビジョン§4.2と整合。

## アーキテクチャ(二層)

```
[リアルタイム層・ローカル・<50ms]                    [内省層・オフライン・Claude]
 既存VADフレーム(16kHz/512)─┐                        ・相槌の語彙/型/ルールを設計(キャラ口調)
 Silero VAD prob ──────────┤                        ・(任意)極小モデルの合成学習データ生成
                            ▼                        ・評価ループ(プライバシー安全な特徴のみ)
                  [BackchannelEngine](純粋ロジック)   ※リアルタイムには居ない
                    発話継続→短い言いよどみ(280ms)を検出
                    + 頻度ガバナ + 反復回避
                            │ BackchannelDecision
                            ▼
                  [selectBackchannel](語プール)
                            │
                            ▼
                  renderer: 事前合成WAV/即時合成 再生 ＋ 非言語アニメ(うなずき)
```

- **WHEN(タイミング)**=既存 `VadSegmenter` と同じ純粋ロジックの状態機械。発話確率列から「持続発話の後の短い言いよどみ(turn-endより手前)」を検出して発火。
- **WHAT(語)**=`characters/{id}/backchannels.json` に外出し(§4.5)。型(continuer/understanding/surprise/empathy)＋thinkingFiller。Phase A は continuer 既定、韻律による型選択は Phase B+。
- **HOW**=事前合成WAVキャッシュ or 即時合成(2-3モーラは速い)＋非言語アニメ。

### 思考フィラー(Phase C・「うーん」)
- 答える入りで熟考時のみ「うーん/そうだなぁ」を出す(=F-ANIM-04「考える間」の音声ツイン)。
- 発火条件は **`optimization-backlog.md` B-15 の「深い/浅い判別」と共有**(雑談は付けない)。`backchannels.json` の `thinkingFiller` は実装済み(プールのみ)。
- 相槌(聞くターン)とは別装置。三原則の「判別テスト(一瞬でも要るか)」を満たす範囲で。

### 層3 暗黙フィードバック(タイミング/頻度の自動微調整)
- **狙い**: 相槌の「着地」を会話の流れから観測し、タイミング(`pauseTriggerMs`)・頻度(`minIntervalMs`)を自動調整(ラベル不要)。今は閾値が良好なので**堅牢化/保険**。
- **信号**: 相槌発火直後のユーザー挙動 = 即再開(〜200ms)→被った(ネガ) / 自然な間のあと継続→良(ポジ) / そのまま沈黙(turn-end)→中立〜軽マイナス。衝突率の移動平均で評価。
- **調整**: 衝突率が高ければ pauseTrigger/minInterval を**少し上げる**(逆は慎重に下げる)。**ハード上下限・1回数ms・数十回ごと**のゆっくり制御。学習値は Lv2b の calibration へ相乗りで永続化。
- **🔴必須ガード(エコー)**: ENE 自身の相槌音がマイクに回り込むと「即再開」を誤検知(偽の衝突・N-17-9 同根)。**相槌の再生中の窓を判定から除外**する(合成WAV長は既知。`setVadSpeaking` 系の仕組みを相槌にも短時間適用 or 時刻窓で除外)。これを外すと学習が壊れる。
- **スコープの正直さ**: 学べるのは**タイミング/頻度のみ**。相槌の**型の適切さ**(うん/へえ)は暗黙信号が無いので学べない(=Lv2 韻律 or 層4 内容理解 or 明示フィードバックの領域)。
- **足がかり**: `VadRuntime` が相槌送出時刻＋音声長を記録 → 次の speech-start までの間隔を測る → `BackchannelController` が衝突率を更新し閾値を調整。

### 層4 オフライン評価ループ(Claude=教師・教師蒸留)
- **狙い**: 構造的改善(新しい型・内容理解=Level3・制御方針の見直し)を**開発時に Claude が**担う。今セッションで私が手で「ログを読んで閾値調整」したことの半自動版。
- **素材**: エンジンが貯める**プライバシー安全な数値トレース**(cue/比/閾値/衝突率の分布)。**生音声・本文は一切出さない(§4.2/§6.2)**。合成データ＋ユーザーが選んだテストセッションのみ。
- **教師蒸留**: ルールが頭打ちになったら、Claude が「良い相槌タイミング/型」のラベル/合成データを生成 → **極小ローカルモデルへ蒸留**(本CPU・調教不要)。リアルタイムには Claude を置かない。
- **止め時**: 人間判定(成功基準8=ユーザーの耳)。"無限自己進化"ではなく「あなたが頷くまで」。

### Phase D 受入・ドキュメント反映
- **人間判定**: 一定期間の実利用での自然さ(成功基準8)。
- **ドキュメント反映(保留中)**: `02_requirements` に **F-LISTEN-xx**、`03_design` §2(新規ソース/`backchannels.json`/`backchannel-calibration.json`)・§11(能動性/相槌エンジン)、`implementation-notes` に N-18-x。→ **ユーザーのドキュメント大改修と衝突するため、それが落ち着いてから反映**。

## 受入チェックリスト

### 自動(vitest / typecheck)
- [ ] `BackchannelEngine`:発話継続なしでは発火しない/最小発話後の言いよどみで発火/頻度ガバナ(連続発火しない)/turn-end手前でのみ発火/reset
- [ ] `selectBackchannel`:型に応じた語を返す/直前と同じ語を避ける/空プールはフォールバック
- [ ] 既存テストが回帰しない

### リッスン専用テストモード(レイテンシ問題と切り離して相槌だけ検証・N-18-2)
- `ENE_LISTEN_ONLY=1` で `npm run dev` を起動すると、ハンズフリーのターン終了時に**文字起こし・応答(Claude/記憶=レイテンシ源)をスキップ**し、VAD＋相槌だけが動く。
  - PowerShell: `$env:ENE_LISTEN_ONLY='1'; npm run dev`
  - 相槌音声には AivisSpeech 起動＋`voice.json`＋`backchannels.json` が必要。**うなずきだけなら Silero VAD のみ**(Whisper/Claude 不要)。
  - 既定 OFF・本番無影響(env 未設定なら通常動作)。
- 検証後は env を外して(または別シェルで)通常起動に戻す。

### 手動(ユーザー・人間判定=成功基準8)
- [ ] 相槌のタイミングが自然(早すぎ/打ちすぎでない)
- [ ] 「聞いてもらえている」感が出る
- [ ] 「うーん」が**熟考時のみ**で全ターンに付かない
- [ ] 声・間がキャラらしい

## やってはいけないこと

- ❌ リアルタイム経路に Claude/ネットワークを置く(往復遅延・プライバシー・ユーザのターン侵害)
- ❌ 相槌の尺・有無を「Claudeが返るまでの時間」で決める(=遅延隠しに堕ちる・三原則違反)
- ❌ 性格を言い訳に「遅い」を正当化する
- ❌ 実会話の生音声・本文を外部へ出す(内省層は合成/抽象特徴/ユーザーが選んだテストのみ・§4.2/§6.2)
- ❌ ユーザーに調教を要求する設計(学習は Claude が本CPUで・ユーザーは試聴のみ)
- ❌ 新規 npm ライブラリの無断追加(§2.3)/保存される数値感情状態(§5.3)
- ❌ コアを重くする(超軽量・ローカル維持・§4.3)
